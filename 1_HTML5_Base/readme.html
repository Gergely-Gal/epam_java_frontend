HTML stands for HyperText Markup Language. Kind of a tautology, but it's a markup language describing web documents.

Markup of an HTML page

DOCTYPE declaration

All HTML documents MUST start with a DOCTYPE declaration. Omitting it is considered an error, and the browser enters into quirks mode.

The DOCTYPE declaration tells the user agent (typically the browser) what kind of document is being rendered.

Html tag

After DOCTYPE, your page should have an HTML tag. It describes the web page.

Notice the word tag above. HTML is built from a tree of tags - called the Document Object Model or DOM. These tags, forming a tree, are defining the markup and semantic meaning of your document.

Head tag

The head tag stores metadata about your document, such as its title, information for search engines, etc. Note that this metadata is not displayed on your page. The head can also include style and script references.

The head tag can contain the following types of tags:

Meta

Used for providing metadata about the site. It can have various attributes, such as http-equiv, which is interpreted as an HTTP header.

Meta is also used for SEO (e.g. keywords, description content), and social media (OG protocol).

Title

The title attribute can contain only text, and is displayed in the title bar, tabs, etc. It should show the title of the document.

Stylesheets

Stylesheets can be included using the <link rel="stylesheet" href="style.css"> fragment.

You can also use inline styles using the style tag.

Script
You can include inline scripts or script file references by using the script tag. However, script tags are not restricted to the head section. You can use them anywhere in an HTML document.

Body tag

The body tag is the visible content of your page. It can contain zero or more HTML tags. The rest of this page shows what are the most common HTML tags.

Emphasis

There are several tags in HTML to put emphasis on something, here they are as follows:

b
The HTML <b> Element represents a span of text stylistically different from normal text, without conveying any special importance or relevance.
strong
Strong is used to give text strong importance, so it's not just a stylistic difference.
i
The i element is used to differentiate text from the surrounding contents, like a name of something, a foreign word, etc.
em
The em tag is used to give some importance, or emphasis to its contents.

Lists

There are three types of lists in HTML. ul represents an unordered, ol an ordered and dl a description list.

Adding list items is done by the li element.

An unordered list example:

  • Item 1
  • Item 2

An ordered list example:

  1. Item 1
  2. Item 2

A description list example:

Item 1
Item 1 is about...
Item 2
Item 2 is about...

divs

div or the division element is used to slice up a page without any semantic information. Essentially it's just a content holder.

If you have an element with a specific task, use a semantic equivalent instead. E.g. if you have a footer, use the footer semantic tag instead of a div.
